# 宝塔面板部署指南

## 一、准备工作

### 1.1 需要上传的文件
将以下文件打包成 zip：
```
scada-system.zip
├── server.ts                    # 后端主文件
├── db.config.js                 # 数据库配置
├── package.json                 # 依赖配置
├── package-lock.json            # 依赖锁定
├── .env.local                   # 环境变量
├── dist/                        # 前端打包文件
│   ├── index.html
│   └── assets/
└── src/
    └── mqttClient.ts           # MQTT 客户端
```

## 二、宝塔面板部署步骤

### 2.1 安装 Node.js 环境

1. **登录宝塔面板**
   - 访问：http://服务器IP:8888
   - 输入用户名和密码

2. **安装 Node.js 版本管理器**
   - 点击左侧菜单 `软件商店`
   - 搜索 `Node 版本管理器`
   - 点击 `安装`
   - 等待安装完成

3. **安装 Node.js 18.x**
   - 安装完成后，点击 `设置`
   - 选择 `版本管理`
   - 点击 `安装` Node.js 18.x 版本
   - 等待安装完成

### 2.2 上传项目文件

1. **创建项目目录**
   - 点击左侧菜单 `文件`
   - 进入 `/www/wwwroot/` 目录
   - 点击 `新建目录`，输入 `scada-system`
   - 进入 `scada-system` 目录

2. **上传文件**
   - 点击 `上传`
   - 选择你打包好的 `scada-system.zip`
   - 等待上传完成
   - 右键点击 zip 文件，选择 `解压`
   - 解压完成后，删除 zip 文件

3. **最终目录结构**
   ```
   /www/wwwroot/scada-system/
   ├── server.ts
   ├── db.config.js
   ├── package.json
   ├── package-lock.json
   ├── .env.local
   ├── dist/
   └── src/
   ```

### 2.3 配置环境变量

1. **编辑 .env.local 文件**
   - 在文件管理中找到 `.env.local`
   - 点击 `编辑`
   - 修改内容：

```env
# 使用生产环境数据库
DB_ENV=production

# 生产环境 SQL Server 配置
DB_PROD_HOST=192.168.4.54
DB_PROD_PORT=1433
DB_PROD_USER=sa
DB_PROD_PASSWORD=zdzn@2026
DB_PROD_NAME=djycs
```

   - 点击 `保存`

### 2.4 安装项目依赖

1. **打开终端**
   - 点击左侧菜单 `终端`
   - 或者点击右上角的 `终端` 图标

2. **进入项目目录**
   ```bash
   cd /www/wwwroot/scada-system
   ```

3. **安装依赖**
   ```bash
   npm install
   ```
   
   等待安装完成（可能需要几分钟）

4. **安装全局工具**
   ```bash
   npm install -g tsx pm2
   ```

### 2.5 创建 Node.js 项目（重要！）

1. **在宝塔面板创建 Node 项目**
   - 点击左侧菜单 `网站`
   - 点击 `Node项目`
   - 点击 `添加Node项目`

2. **填写项目信息**
   ```
   项目名称：SCADA监控系统
   项目路径：/www/wwwroot/scada-system
   启动文件：server.ts
   运行端口：3000
   Node版本：选择 18.x
   启动方式：选择 pm2
   ```

3. **重要：添加环境变量**
   在"环境变量"部分添加：
   ```
   NODE_ENV=production
   ```
   这个非常重要！它告诉后端使用生产模式，提供静态文件。

4. **点击 `提交`**

### 2.6 启动项目

项目创建后会自动启动，你也可以手动控制：

1. **在 Node项目 列表中**
   - 找到 `SCADA监控系统`
   - 点击 `启动` 按钮
   - 状态变为 `运行中` 表示成功

2. **查看日志**
   - 点击项目名称或 `日志` 按钮
   - 查看启动日志，应该看到：
   ```
   ✅ MySQL 连接池已创建（本地环境）
   或
   ✅ SQL Server 连接成功（线上环境）
   ✅ 数据库初始化成功
   ✅ 服务器 MQTT 已连接
   🚀 服务器启动成功！
   📍 访问地址: http://localhost:3000
   ```

### 2.7 配置防火墙

1. **开放 3000 端口**
   - 点击左侧菜单 `安全`
   - 点击 `添加端口规则`
   - 填写：
     ```
     端口：3000
     协议：TCP
     策略：允许
     备注：SCADA系统
     ```
   - 点击 `确定`

### 2.8 访问应用

在浏览器中访问：
```
http://服务器IP:3000
```

## 三、配置域名访问（可选）

### 3.1 添加网站

1. **创建网站**
   - 点击左侧菜单 `网站`
   - 点击 `添加站点`
   - 填写：
     ```
     域名：scada.yourdomain.com
     根目录：/www/wwwroot/scada-system/dist
     PHP版本：纯静态
     ```
   - 点击 `提交`

### 3.2 配置反向代理

1. **设置反向代理**
   - 在网站列表中找到刚创建的网站
   - 点击 `设置`
   - 点击 `反向代理`
   - 点击 `添加反向代理`

2. **填写代理信息**
   ```
   代理名称：SCADA-API
   目标URL：http://127.0.0.1:3000
   发送域名：$host
   ```

3. **配置规则**
   - 点击 `配置文件`
   - 找到 `location /` 部分
   - 修改为：

```nginx
location / {
    root /www/wwwroot/scada-system/dist;
    try_files $uri $uri/ /index.html;
}

location /api {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header REMOTE-HOST $remote_addr;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}

location /mqtt {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "Upgrade";
    proxy_set_header Host $host;
}
```

   - 点击 `保存`

4. **访问**
   ```
   http://scada.yourdomain.com
   ```

## 四、数据库配置

### 4.1 如果使用宝塔的 MySQL

1. **安装 MySQL**
   - 点击 `软件商店`
   - 搜索 `MySQL`
   - 安装 MySQL 5.7 或 8.0

2. **创建数据库**
   - 点击左侧菜单 `数据库`
   - 点击 `添加数据库`
   - 填写：
     ```
     数据库名：jiankong_zhongdei
     用户名：root
     密码：123456
     ```
   - 点击 `提交`

3. **修改 .env.local**
   ```env
   DB_ENV=local
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_USER=root
   DB_PASSWORD=123456
   DB_NAME=jiankong_zhongdei
   ```

4. **重启项目**
   - 在 Node项目 中点击 `重启`

### 4.2 如果使用远程 SQL Server

保持 `.env.local` 配置不变：
```env
DB_ENV=production
DB_PROD_HOST=192.168.4.54
DB_PROD_PORT=1433
DB_PROD_USER=sa
DB_PROD_PASSWORD=zdzn@2026
DB_PROD_NAME=djycs
```

## 五、常用操作

### 5.1 查看项目状态

1. **在宝塔面板**
   - 点击 `网站` → `Node项目`
   - 查看项目状态（运行中/已停止）

2. **查看日志**
   - 点击项目名称
   - 查看实时日志

### 5.2 重启项目

1. **在 Node项目 列表**
   - 点击 `重启` 按钮

2. **或使用终端**
   ```bash
   pm2 restart scada-system
   ```

### 5.3 停止项目

1. **在 Node项目 列表**
   - 点击 `停止` 按钮

2. **或使用终端**
   ```bash
   pm2 stop scada-system
   ```

### 5.4 查看日志

1. **在宝塔面板**
   - Node项目 → 点击项目名称 → 查看日志

2. **或使用终端**
   ```bash
   pm2 logs scada-system
   ```

3. **查看 MQTT 数据**
   ```bash
   cat /www/wwwroot/scada-system/mqtt_latest_data.json
   ```

### 5.5 更新项目

1. **停止项目**
   - 在 Node项目 中点击 `停止`

2. **备份当前版本**
   - 在文件管理中，右键 `scada-system` 目录
   - 选择 `压缩`
   - 命名为 `scada-system-backup-日期.zip`

3. **上传新文件**
   - 删除旧文件（保留 node_modules 和 .env.local）
   - 上传新的文件
   - 解压

4. **重新安装依赖（如果 package.json 有变化）**
   ```bash
   cd /www/wwwroot/scada-system
   npm install
   ```

5. **重启项目**
   - 在 Node项目 中点击 `启动`

## 六、故障排查

### 6.1 项目无法启动

**查看日志：**
- Node项目 → 点击项目名称 → 查看错误日志

**常见问题：**

1. **端口被占用**
   ```bash
   # 查看端口占用
   netstat -tuln | grep 3000
   
   # 杀死占用进程
   lsof -ti:3000 | xargs kill -9
   ```

2. **依赖未安装**
   ```bash
   cd /www/wwwroot/scada-system
   npm install
   ```

3. **Node 版本不对**
   - 在 Node项目 设置中，确保选择的是 18.x 版本

### 6.2 数据库连接失败

1. **检查数据库服务**
   - 点击 `数据库`
   - 确保 MySQL 服务正在运行

2. **检查配置**
   - 编辑 `.env.local`
   - 确认数据库地址、端口、用户名、密码正确

3. **测试连接**
   ```bash
   mysql -u root -p -h 127.0.0.1
   ```

### 6.3 MQTT 连接失败

1. **检查网络**
   ```bash
   telnet 106.12.13.32 8083
   ```

2. **检查防火墙**
   - 确保服务器可以访问外网
   - 检查宝塔安全规则

### 6.4 前端无法访问

1. **检查端口**
   - 安全 → 确保 3000 端口已开放

2. **检查项目状态**
   - Node项目 → 确保项目在运行中

3. **检查 dist 目录**
   - 文件管理 → 确保 dist 目录存在且有内容

## 七、性能优化

### 7.1 开启 Gzip 压缩

在网站设置中：
- 点击 `配置文件`
- 找到 `gzip` 相关配置
- 确保开启：
  ```nginx
  gzip on;
  gzip_types text/plain text/css application/json application/javascript;
  ```

### 7.2 设置静态资源缓存

在网站配置文件中添加：
```nginx
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

### 7.3 配置 SSL 证书（推荐）

1. **申请 SSL 证书**
   - 网站设置 → SSL
   - 选择 `Let's Encrypt`
   - 点击 `申请`

2. **强制 HTTPS**
   - 开启 `强制HTTPS` 开关
